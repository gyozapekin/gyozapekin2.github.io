8締切設定の月別＆個別調整（Ver.3.0新機能）

スタッフが何月分のシフト希望を出せるかをコントロールします。デフォルトは前月の20日。月別・スタッフ別に上書き可能。

階層構造

締切は次の順に判定されます（上ほど優先）：

  1. 個人別の上書き（特定スタッフだけ別の締切）
  2. 月別の上書き（その月全体の締切を変更）
  3. デフォルト（前月の20日 23:59）

月別の調整

各月の行で日付ピッカーを変更→「保存」。元に戻すには「リセット」ボタン。

個別調整

月別の右にある個別ボタンで展開。スタッフごとに別の締切日を設定できます。

「変更なし」は保存しても上書きが増えません。 うっかり開いて「保存」を押しても何も書き込まれない設計（Ver.3.0で導入）。値を変えた時だけ上書きとして記録されます。

使用例

  • 連休・繁忙月に「全員の締切を5日延ばす」→ 月別の日付ピッカーで変更→保存
  • 特定スタッフだけ「旅行から戻った日まで延長」→ 個別パネルでその人だけ変更
  • 普段通りに戻したい → 「リセット」

9テンプレート（A週／B週）

テンプレート画面（A週／B週の2週ローテーション）
  • A週は1・3・5週目、B週は2・4週目に使われます
  • 各曜日の昼／夜にスタッフを設定（複数可）
  • 「今月に反映」で表示月の下書きをテンプレートから再生成
  • 定休日は自動で「定休」表示になる

10統計で偏りをチェック（Ver.3.0新機能）

1コマ＝4時間で計算します。シフト管理を変更すると即座に再計算されます。

対象切替

下書き（編集中） 公開済み

どちらを集計するか選択。デフォルトは「下書き」です。

サマリー

項目内容
営業日数定休日と臨時休業を除いた当月の営業日
昼／夜コマ合計1日=最大2コマ（昼1＋夜1）
合計人時全スタッフの合計勤務時間（コマ×4h）
出勤スタッフ数当月にシフトが入っているスタッフ数

スタッフ別表

各スタッフの昼／夜コマ数、合計時間、占有率%、最長連勤日数を表示。

連勤警告：3連勤以上は「⚠」、5連勤以上は「⚠⚠」が表示されます。定休日はカウントされません（営業日のみで連続判定）。

11シフトを公開する（公開フロー）

ベストプラクティス

  1. テンプレートで下書き作成（テンプレート→「今月に反映」）
  2. シフト管理で調整（編集モード→候補から選んで確定／外す）
  3. 統計で偏りチェック（連勤・占有率を確認）
  4. 公開プレビューでスタッフ視点を確認
  5. 「公開する」を押して確認ダイアログでOK
  6. スタッフへLINE等で「公開しました」と通知

後で修正が必要になったら

編集モード→修正→もう一度「公開する」で再公開（上書き）。スタッフ画面にも自動反映。

未公開に戻すケース

  • 大幅な修正が必要で、いったん全部見直したい時
  • 誤って公開した時

「未公開に戻す」を押すと、スタッフ画面では「まだ公開されていません」表示に戻りますが、下書きは残ったままです。

12リアルタイム同期と編集ロック（Ver.3.0新機能）

リアルタイム同期

同じ画面を複数の端末で開いていても、Firebase経由で自動同期されます。誰かがシフトを変更すると、他の端末にも即座に反映され、画面下にトースト通知が出ます。

編集ロック（排他制御）

  • 1人が編集モードに入ると、他の管理者は閲覧モードになります
  • 30秒以上反応がない場合、「引き継ぐ」ボタンで強制的に編集権を奪取できます
  • 編集中のタブを閉じる（ブラウザ閉じ）と自動的にロック解放

13セキュリティ・トラブルシュート

パスワードの保存

パスワードはSHA-256＋個別ソルトでハッシュ化されてFirebaseに保存されます。平文では保存されません。

Firebase認証

すべての読み書きは Anonymous Auth トークンが必要です（auth != null）。未認証アクセスは Firebase Rules で拒否されます。

環境分離

環境URLFirebase Namespace
本番gyozapekin.com/shift//pekin/db/
Devgyozapekin.com/gyozapekin2.github.io/shift//pekin/db_dev/
本番とDevはFirebaseのネームスペースで完全に分離されています。Devでのテストが本番に影響することはありません。

よくある質問

再同期してもデータが古い
一度ログアウト→再ログイン。それでも改善しない場合、ブラウザのキャッシュをCtrl+Shift+Rで強制リロード。
公開ボタンを押しても反映されない
F12でブラウザの開発者ツールを開き、ConsoleタブでFirebaseエラーが出ていないか確認。「permission denied」が出る場合はFirebase RulesまたはAuth設定の問題。
スタッフが「受付終了」で希望を出せない
設定→「シフト希望締切設定」で該当月の締切を延長。または個別パネルでそのスタッフだけ延長。
編集モードに入れない
他の管理者がロック中。30秒以上経過していれば「引き継ぐ」ボタンが出ます。それでも入れない場合、相手の端末でブラウザを閉じてもらってください。
スタッフのパスワードを忘れたと言われた
スタッフ管理タブで該当スタッフを「編集」→パスワードを新規入力。マスターパスワード機能を設定しておけば、スタッフは自分で予備パスワードでログインできます。
テンプレート反映を間違えて押した
編集モードに入って「戻る」で取消。または再度テンプレートを修正→「今月に反映」で正しい状態を再生成。
公開済みのシフトを修正したい
編集モード→修正→もう一度「公開する」で上書き再公開。スタッフ画面に即時反映されます。
Dev環境で操作した内容が本番に出てしまった
原理的に起こりえません。Firebaseネームスペースが分離されています。もし本当に起きていたら、URL を再確認してください（gyozapekin2.github.io が含まれているか）。
困ったときはグループラインで連絡してください。
