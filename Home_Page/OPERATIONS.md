---

## 新しい記事を投稿する

### 手順（GitHub Web UI / スマホからでも可能）

1. GitHubにログイン
2. リポジトリ `gyozapekin/gyozapekin.github.io` を開く
3. `_posts` フォルダをクリック
4. 右上「Add file」→「Create new file」をクリック
5. ファイル名を `YYYY-MM-DD-shortname.md` の形式で入力
   - 例: `2026-05-01-golden-week.md`
   - 日付必須、タイトル用の英数字部分はアルファベット＋ハイフン
6. 下記テンプレートをコピペして編集
7. 右下「Commit changes」ボタンで保存 → 約1〜2分後に自動公開

### 記事テンプレート

```markdown
---
layout: post
title: "ここに記事のタイトル"
date: 2026-05-01 12:00:00 +0900
news_category: oshirase
description: "検索結果やSNSシェアに表示される短い説明文（120文字以内推奨）"
---

ここから本文を書きます。

## 見出し2

段落を分けるには、このように**空行**を入れてください。

テキストを **太字** にしたり、[リンクを貼る](https://example.com) こともできます。

### 箇条書き

- 項目1
- 項目2
- 項目3

### 画像の挿入

![画像の説明](/images/photo-2026-05-01.jpg)
```

### Markdownの基本書き方

| 書き方 | 結果 |
|---|---|
| `# 見出し1` | 大見出し |
| `## 見出し2` | 中見出し |
| `### 見出し3` | 小見出し |
| `**太字**` | **太字** |
| `*斜体*` | *斜体* |
| `- 項目` | 箇条書き |
| `1. 項目` | 番号リスト |
| `[リンク文字](URL)` | リンク |
| `![alt](画像URL)` | 画像 |
| `> 引用` | 引用 |
| `---` | 区切り線 |

## よくある質問

### Q. 日本語のファイル名で作ってしまった
→ ファイル名は半角英数字とハイフンのみ。日本語は公開不可。

### Q. 画像はどこに入れますか？
→ `images/` フォルダへアップロード、記事内は `![alt](/images/ファイル名.jpg)` で参照。

### Q. スマホから投稿できますか？
→ はい。GitHub公式アプリまたはブラウザから、同じ手順で編集可能。

### Q. 記事を削除したい
→ `_posts/` 内のファイルを GitHub Web UI から削除。URL変更はSEO的にリスクあり。
